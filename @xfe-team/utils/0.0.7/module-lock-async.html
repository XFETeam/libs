lock-async

(require("lock-async"))(uniqueNameopt) → {function}

Source:
  • lock-async.js, line 33

锁定异步请求装饰器 开发者可以通过 lockAsync 装饰器确保在 async 结束过程中不会有二次调用, 即防止一个按钮在"没完成"时被多次点击

Example
使用方式1:

class {
  @lockAsync()
  async onXXXClick() {
    await xxx.getUserInfo()
  }
}

使用方式2:
const locker = lockAsync();
locker(<函数>)
Parameters:
Name Type Attributes Description
uniqueName string <optional>

锁名, 一个锁名代表着一把锁, 开发者可以自定义使用同一个锁名去控制所有的锁, 即当一个页面中都使用一个锁名时, 一个按钮触发的异步请求, 所有按钮都可以被锁定.

当不传入锁名时, 该装饰器会为每一个装饰函数设置一个唯一的锁, 即可以避免同一个按钮在"异步状态"未完成时候被多次点击
